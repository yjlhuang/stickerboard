## 進行中 / 卡點

**目前沒有進行中的開發項目。** 沒有已知的 bug 或阻塞。

唯一長期存在的「非阻塞限制」：整個工具是單一 HTML 檔案、無 build step，
所以不能用 ES module（`file://` 下會被 CORS 擋掉，破壞「雙擊即用」）。
這是刻意的取捨，不是待修項目。

## 長期構想

> ⚠️ 以下都**只是構想，尚未承諾、沒有時程**。

完整藍圖寫在 [`stickerboard architecture notes.md`](../stickerboard%20architecture%20notes.md)，這裡只列標題，細節不重複：

- **Product B：LINE Helper**——LINE Creators Market 用的貼圖包工具，獨立 HTML，
  **不與 StickerBoard 的 UI 合併**（兩者的需求完全不同：一個要創作自由，一個要規格一致）。
  兩條產品線只共用 Prompt Engine（`shared/recipe-data.js` + `shared/recipe.js`）。
- **Typography v2**：角色偏好、Pack Mode、系列生成器
- **Typography v3**：LINE Helper 整合、批量貼圖、ZIP 匯出

**核心原則（藍圖裡最重要的一句）**：重點不是增加更多預設模板，而是維持一套可重複使用的
Prompt Engine——Emotion 管情緒、Aesthetic 管風格、Route 管生成策略，三者保持解耦。
